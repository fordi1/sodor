# سند معماری نرم‌افزار اپلیکیشن حرفه‌ای فاکتور و پیش‌فاکتور (فاکتور دفتری)
## Software Architecture Document (SAD)

این سند توصیف‌کننده معماری نرم‌افزاری اپلیکیشن اندرویدی آفلاین-محور **«فاکتور دفتری»** به زبان فارسی (RTL) است.

---

## ۱. لیست قابلیت‌ها (Features List)

اپلیکیشن **فاکتور دفتری** به صورت کاملاً آفلاین (Offline-First) طراحی می‌شود و تمام داده‌ها در حافظه محلی دستگاه کاربر با امنیت بالا ذخیره می‌گردند.

### الف) مدیریت فاکتورها و پیش‌فاکتورها (Invoices & Proformas)
- **دو حالت کاری متفاوت**: ساخت فاکتور رسمی/غیررسمی و پیش‌فاکتور با امکان تبدیل سریع پیش‌فاکتور به فاکتور نهایی.
- **فرمت شماره‌گذاری سفارشی**: خودکار یا دستی بر اساس الگوهای مشخص (مانند `INV-1405-001`).
- **محاسبات دقیق و پیشرفته**:
  - محاسبه قیمت کل اقلام به صورت زنده.
  - اعمال تخفیف خطی (برای تک‌تک اقلام) یا تخفیف کلی فاکتور (به درصد یا مبلغ ثابت).
  - محاسبه مالیات بر ارزش افزوده (VAT) مطابق قوانین مالیاتی کشور (قابل سفارشی‌سازی یا غیرفعال‌سازی).
  - محاسبه هزینه ارسال یا خدمات فرعی.
- **مدیریت وضعیت فاکتور**:پیش‌نویس (Draft)، پرداخت‌نشده (Unpaid)، پرداخت‌شده (Paid)، باطل‌شده (Cancelled).
- **امضای فاکتور**: امکان افزودن تصویر امضا یا ثبت امضای دیجیتال زنده روی صفحه گوشی برای درج در فاکتور.

### ب) خروجی و اشتراک‌گذاری (PDF & Printing)
- **خروجی PDF استاندارد با پشتیبانی کامل از فونت فارسی**: تولید فاکتور شکیل آ۴ (A4)، آ۵ (A5) و فیش ۸۰ میلی‌متری حرارتی (Thermal Printer).
- **اشتراک‌گذاری آسان**: ارسال فایل PDF از طریق شبکه‌های اجتماعی ایرانی و خارجی (ایتا، بله، واتساپ، تلگرام و غیره).
- **چاپ مستقیم (Google Print Web)**: اتصال به چاپگرهای شبکه یا بلوتوثی برای چاپ سریع برگه‌ها.

### ج) مدیریت مشتریان (Customer Directory)
- ثبت اطلاعات کامل مشتری شامل نام حقیقی/حقوقی، شماره تماس، کد ملی/شناسه ملی، کد اقتصادی، آدرس پستی و ایمیل.
- تاریخچه خرید هر مشتری با تفکیک بدهی‌ها و خریدهای تسویه شده.
- دسترسی سریع به شماره تماس جهت تماس مستقیم از درون برنامه.

### د) مدیریت کالاها و خدمات (Product & Service Catalog)
- ثبت کالاها/خدمات به همراه نام، شرح، قیمت پیش‌فرض، نوع واحد فروش (عدد، کیلوگرم، متر، ساعت و غیره).
- بارکدخوان بومی (Native Barcode Scanner) با دوربین جهت افزودن کالاها به فاکتور به سرعت بالا.
- دسته‌بندی کالاها و مدیریت موجودی پایه در حالت آفلاین.

### ه) آمار و داشبورد مدیریتی (Dashboard & Insights)
- نمودارهای درآمد، تعداد فاکتورها، حجم فروش ماهانه و سالانه (بومی با Jetpack Compose Canvas به صورت آفلاین).
- نمایش سریع فاکتورهای سررسید شده و بدهی‌های معوقه مشتریان.
- کالاهای پرفروش و پرسودترین مشتریان.

### و) تنظیمات کسب‌وکار و پشتیبان‌گیری (Business Setup & Backup)
- پروفایل کسب‌وکار شامل نام، لوگو، کداقتصادی، تلفن، آدرس، کارت بانکی لیدر و شماره شبا جهت نمایش در انتهای فاکتور.
- **پشتیبان‌گیری آفلاین**: خروجی صادر شده از دیتابیس در قالب یک فایل رمزگذاری شده به صورت محلی در حافظه گوشی کاربر یا خروجی اکسل (Excel Export) از لیست فاکتورها.

## ۲. نقش‌های کاربران (User Roles)

با توجه به آفلاین و محلی (Single-Device Tracker) بودن اپلیکیشن، سیستم بر پایه یک نقش اصلی طراحی شده است:

1. **مدیر کسب‌وکار (Business Owner / Administrator / Solo-User)**:
   - دارای دسترسی کامل به تمامی بخش‌های نرم‌افزار.
   - امکان تعریف کالاها، مشتریان، ایجاد و حذف فاکتورها، و مدیریت تنظیمات کسب‌وکار خود.
   - فرآیند احراز هویت زیست‌سنجی (Biometric Authentication - اثر انگشت) یا رمز عبور عددی جهت امنیت اطلاعات در صورت تمایل کاربر.

## ۳. طراحی دیتابیس (Room Database Design)

دیتابیس سیستم با استفاده از **Room Database** به صورت یکپارچه و با رعایت روابط دقیق بین جداول پیاده‌سازی خواهد شد.

### موجودیت‌ها و فیلدهای هر جدول (Schema)

#### ۱) جدول مشخصات کسب‌وکار (`business_profile`)
جهت نگهداری اطلاعات هویتی و مالی صادرکننده فاکتور.

| نام فیلد (Kotlin) | نوع داده (SQLite) | توضیحات |
| :--- | :--- | :--- |
| `id` (PK) | INTEGER | شناسه یکتا (همواره مقدار ثابت ۱ دارد) |
| `businessName` | TEXT | نام فروشگاه یا شرکت |
| `ownerName` | TEXT | نام صاحب کسب‌وکار |
| `economicCode` | TEXT | کد اقتصادی |
| `nationalId` | TEXT | شناسه ملی یا کد ملی |
| `phoneNumber` | TEXT | شماره تماس ثابت یا همراه |
| `address` | TEXT | آدرس کامل |
| `postalCode` | TEXT | کد پستی |
| `logoPath` | TEXT | آدرس محلی ذخیره عکس لوگو روی دستگاه |
| `signaturePath` | TEXT | آدرس محلی ذخیره عکس امضا/مهر روی دستگاه |
| `bankName` | TEXT | نام بانک جهت واریز |
| `cardNumber` | TEXT | شماره کارت بانکی |
| `shabaNumber` | TEXT | شماره شبا |

#### ۲) جدول مشتریان (`customers`)
| نام فیلد (Kotlin) | نوع داده (SQLite) | توضیحات |
| :--- | :--- | :--- |
| `id` (PK) | INTEGER | تولید خودکار (AutoGenerate) |
| `fullName` | TEXT | نام و نام خانوادگی یا نام شرکت (نمایه‌سازی شده جهت جستجو سریع) |
| `phoneNumber` | TEXT | شماره تماس اصلی مشتری |
| `nationalId` | TEXT | کد ملی یا شناسه ملی |
| `economicCode` | TEXT | کد اقتصادی مشتری (برای فاکتورهای رسمی) |
| `address` | TEXT | آدرس مشتری |
| `email` | TEXT | ایمیل مشتری |
| `postalCode` | TEXT | کد پستی مشتری |
| `createdAt` | INTEGER | زمان ثبت به میلی‌ثانیه |

#### ۳) جدول کالاها و خدمات (`products`)
| نام فیلد (Kotlin) | نوع داده (SQLite) | توضیحات |
| :--- | :--- | :--- |
| `id` (PK) | INTEGER | تولید خودکار |
| `name` | TEXT | نام محصول/خدمت |
| `description` | TEXT | توضیحات اختیاری |
| `defaultUnitPrice`| REAL | قیمت واحد پیش‌فرض (Double) |
| `unit` | TEXT | واحد شمارش (مثلا: عدد، کیلوگرم، کارتن، متر) |
| `barcode` | TEXT | کد بارکد کالا جهت اسکن سریع |
| `taxRate` | REAL | درصد مالیات پیش‌فرض برای این کالا |
| `createdAt` | INTEGER | زمان ثبت |

#### ۴) جدول فاکتور اصلی (`invoices`)
| نام فیلد (Kotlin) | نوع داده (SQLite) | توضیحات |
| :--- | :--- | :--- |
| `id` (PK) | INTEGER | تولید خودکار |
| `invoiceNumber` | TEXT | شماره فاکتور منحصر به فرد |
| `customerId` | INTEGER | کلید خارجی متصل به `customers.id` |
| `issueDate` | INTEGER | تاریخ صدور کارت به پیش‌فرض شمسی (ذخیره به میلی‌ثانیه دیتابیس) |
| `dueDate` | INTEGER | تاریخ سررسید فاکتور |
| `invoiceType` | TEXT | نوع فاکتور: `INVOICE` یا `PROFORMA` |
| `status` | TEXT | وضعیت: `DRAFT`، `UNPAID`، `PAID`، `CANCELLED` |
| `discountAmount` | REAL | مبلغ تخفیف کلی روی فاکتور |
| `taxRate` | REAL | نرخ مالیات کلی ارزش افزوده (مثلا ۱۰ درصد) |
| `shippingFee` | REAL | هزینه حمل و نقل یا ارسال |
| `notes` | TEXT | یادداشت‌ها و توضیحات پایانی فاکتور |
| `createdAt` | INTEGER | زمان ثبت |

#### ۵) جدول اقلام فاکتور (`invoice_items`)
ارتباط یک‌به‌چند از فاکتور به کالاها.

| نام فیلد (Kotlin) | نوع داده (SQLite) | توضیحات |
| :--- | :--- | :--- |
| `id` (PK) | INTEGER | تولید خودکار |
| `invoiceId` | INTEGER | کلید خارجی متصل به `invoices.id` (با قابلیت `ON DELETE CASCADE`) |
| `productId` | INTEGER | کلید خارجی متصل به `products.id` (اختیاری، جهت ارجاع به کالا) |
| `productName` | TEXT | کپی نام کالا در لحظه صدور فاکتور (برای جلوگیری از به هم خوردن قیمت‌ها با تغییر کالا) |
| `unitPrice` | REAL | قیمت واحد کالا در فاکتور |
| `unit` | TEXT | واحد کالا |
| `quantity` | REAL | مقدار یا تعداد فروش کالا (Double جهت پشتیبانی از مقادیر اعشاری مثل ۱.۵ کیلوگرم) |
| `discountAmount` | REAL | تخفیف اعمال شده برای این کالا |
| `taxAmount` | REAL | مالیات محاسبه شده برای این کالا |

## ۴. لیست صفحات (Screen List)

طراحی صفحات منطبق با معماری تک‌صفحه‌ای Jetpack Compose و ناوبری Type-Safe می‌باشد. روند نمایش کاملاً RTL و به زبان شیرین فارسی است:

1. **صفحه خانه/داشبورد (DashboardScreen)**:
   - خلاصه کارکرد مالی (آمار کل فروش ماه، فاکتورهای تسویه نشده، نمودار خطی فروش هفته).
   - دسترسی‌های سریع: ایجاد فاکتور جدید، مشتری جدید، اسکنر سریع کالا.
   - ویجت لیست آخرین فاکتورهای سررسید شده.

2. **صفحه لیست فاکتورها (InvoiceListScreen)**:
   - تب‌بندی بین «فاکتورها» و «پیش‌فاکتورها».
   - فیلترهای پیشرفته بر اساس وضعیت پرداخت، بازه زمانی، و جستجوی نام مشتری یا شماره فاکتور.
   - دکمه‌های کنترلی سریع برای اشتراک‌گذاری مستقیم یا تغییر وضعیت تسویه حساب.

3. **صفحه ایجاد/ویرایش فاکتور (InvoiceEditorScreen)**:
   - یک جادوگر گام‌به‌گام (Multi-step or clean single scroll view):
     - **گام ۱**: اطلاعات پایه (تاریخ، نوع فاکتور، انتخاب مشتری از دفتر تلفن).
     - **گام ۲**: انتخاب اقلام (افزودن کالاها، تنظیم تعداد، درج تخفیف هر قلم یا کالا، تغییر قیمت).
     - **گام ۳**: تنظیمات مالی کلی (درصد مالیات، تخفیف کل، هزینه حمل و نقل).
     - **گام ۴**: پیش‌نمایش نهایی محاسبات، افزودن یادداشت و ذخیره‌سازی.

4. **پیش‌نمایش فاکتور و صدور خروجی (InvoiceDetailsScreen)**:
   - نمای بسیار شکیل فاکتور چیده شده با کادرها و امضای فروشگاه به صورت کاملا راست‌چین.
   - دکمه‌های «تولید PDF»، «ارسال در شبکه‌های اجتماعی»، «چاپ فاکتور» و «تغییر وضعیت به تسویه شده».
   - امکان تبدیل پیش‌فاکتور به فاکتور نهایی با یک کلیک.

5. **صفحه دفتر مشتریان (CustomerDirectoryScreen)**:
   - لیست تمام مشتریان ثبت‌شده با قابلیت جستجوی هوشمند عددی و متنی.
   - قابلیت ثبت مشتری جدید یا ویرایش اطلاعات به همراه گزارش فاکتورهای معوقه مشتری.

6. **صفحه کاتالوگ محصولات و خدمات (ProductCatalogScreen)**:
   - لیست موجودی کالاها با قیمت‌ها و واحدها.
   - امکان ثبت سریع کالای جدید به صورت دستی یا با خواندن بارکد کالا.

7. **صفحه تنظیمات و پروفایل کسب‌وکار (SettingsScreen)**:
   - فرم به‌روزرسانی اطلاعات پروفایل فروشگاه شما (آدرس، لوگو، کارت بانکی).
   - دانلود فایل پشتیبان دیتابیس (Backup) یا بازگردانی آن (Restore).
   - انتخاب قالب فاکتور PDF (طرح رسمی مالیاتی، طرح ساده بازاری، طرح فیش چاپگر حرارتی).

## ۵. جریان ناوبری (Navigation Flow)

جریان ارتباطی بین صفحات به صورت زیر تعریف می‌شود:

```
                            [ DashboardScreen ] (ورودی اصلی با آمار)
                                     |
    +-----------------+--------------+---------------+------------------+
    |                 |                              |                  |
[ CustomerList ]  [ ProductCatalog ]          [ InvoiceList ]     [ SettingsScreen ]
    |                 |                              |                  |
[ Add/Edit Cust ] [ Add/Edit Prod ]           [ InvoiceDetails ]  [ Edit Business Profile ]
                                                     |                  |
                                              [ InvoiceEditor ]   [ Backup/Restore ]
```

* **نکات ناوبری**:
  * در کل اپلیکیشن، سیستم به حالت RTL پیکربندی می‌شود.
  * صفحه ویرایش فاکتور پس از اتمام موفق، مستقیماً به صفحه جزئیات فاکتور (InvoiceDetailsScreen) ارجاع داده شده و استک قبلی پاک می‌شود تا از بازگشت مجدد به صفحه نیمه‌کاره جلوگیری شود.

## ۶. پشته فنی پیشنهادی (Technical Stack Recommendation)

- **زبان برنامه‌نویسی**: Kotlin
- **چارچوب رابط کاربری (UI Framework)**: Jetpack Compose (کاملاً بومی، مدرن و با کارایی فوق‌العاده بالا به همراه پیاده‌سازی RTL بر بستر `CompositionLocalProvider(LocalLayoutDirection provides LayoutDirection.Rtl)`).
- **معماری نرم‌افزار**: MVVM (Model-View-ViewModel) منطبق با اصول روز کاتلین و Android Clean Arch.
- **دیتابیس محلی (Database)**: SQLite بر بستر کتابخانه رسمی **Room** به همراه **Kotlin Symbol Processing (KSP)** برای بهینه‌سازی سرعت ساخت اپلیکیشن.
- **برنامه‌نویسی ناهمگام (Asherent Computing)**: Kotlin Coroutines و Kotlin Flow جهت مدیریت واکنش‌گرای جریان داده‌ها از دیتابیس تا لایه UI.
- **مدیریت ناوبری (Navigation)**: Jetpack Navigation Compose به همراه سریال‌سازی نوع‌امن (Kotlinx Serialization).
- **تولید PDF (PDF Generation)**: کتابخانه استاندارد اندروید `android.graphics.pdf.PdfDocument` برای تولید محتوای PDF بومی با بکارگیری فونت‌های رسمی زیبای فارسی نظیر «وزیر» یا «ایران سنس» بدون نیاز به اینترنت و بدون افت کیفیت رندر عکس‌ها.
- **اسکن بارکد (Barcode Scanning)**: استفاده از کیت اسکن بارکد بومی سبک Android CameraX یا Google ML Kit Barcode Scanning بدون هیچ‌گونه نیاز به حافظه زیاد یا مصرف حجم فوق‌العاده اینترنت.

│   │   ├── dao/
│   │   │   ├── BusinessProfileDao.kt
│   │   │   ├── CustomerDao.kt

## ۸. نقشه راه توسعه (Development Roadmap)

رویکرد اجرای توسعه پروژه در ۴ گام مستقل و تست‌پذیر سازماندهی می‌شود:

### فاز ۱: زیرساخت دیتابیس و لایه داده (ثبت مشتریان و محصولات) [زیربنای اصلی]
- پیاده‌سازی دیتابیس Room و رابط کاربری کاتالوگ کالاها و مشتریان.
- فراهم ساختن امکان افزودن، تغییر و جستجوی آسان در هر جدول دیتابیس به صورت آفلاین.
- تمیزسازی کدهای پایه و افزودن فونت زیبای فارسی «وزیر» به کل بدنه برنامه و لایه‌بندی RTL.

### فاز ۲: موتور محاسباتی فاکتور و ایجاد کننده آن [بخش منطقی]
- ساخت لایه ساخت فاکتور (Invoice Editor wizard).
- بررسی درستی روابط محاسباتی (امکان درج خودکار درصد مالیات ارزش افزوده به مبلغ نهایی، تخفیف، اعمال تخفیف جداگانه روی اقلام کالا).
- پیاده‌سازی کلاس هوشمند تبدیل مبالغ عددی به حروف فارسی (جهت جلوگیری از کلاهبرداری و خطاهای خوانایی فاکتور).

### فاز ۳: پیش‌نمایش، امضا و موتور تولید PDF فارسی و چاپ [خروجی برنامه]
- پیاده‌سازی بوم کشیدن امضا به صورت لمسی بر روی دستگاه و ذخیره تصویر آن در دیتابیس.
- توسعه کلاس `PDFGenerator` جهت پیاده‌سازی رندرر بومی PDF فارسی روی سایزهای متداول (A4 و رسید حرارتی سوپرمارکتی).
- فعال‌سازی اشتراک‌گذاری مستقیم از طریق اینتنت اندروید بدون نیاز به ارتباط سروری.

### فاز ۴: آمار داشبورد، بهینه‌سازی و پشتیبان‌گیری [پرداخت نهایی]
- رسم نمودارهای خلاصه آماری زیبا روی داشبورد ورودی با استفاده از Compose Canvas.
- تنظیم قابلیت خروجی اکسل و ثبت فایل پشتیبان از کل دستگاه جهت حفظ امنیت داده‌ها در برابر تعویض گوشی.
- بررسی و دیباگ نهایی.
